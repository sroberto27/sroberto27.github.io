About XR Filmmaking Vault

XR Filmmaking Vault is an open taxonomy, database, and interactive visualization of XR, AR, and VR tools and systems across the film production pipeline. It is designed to support systematic comparison, gap analysis, and tool design exploration for researchers, practitioners, and educators working at the intersection of immersive technology and filmmaking.

This project is modeled after Locomotion Vault (Di Luca, Seifi, Egan & Gonzalez-Franco, CHI 2021), which demonstrated the value of an attribute-based database with similarity-driven navigation for organizing a high-dimensional design space.

Research Framing

Virtual production is widely framed as a way to make film and television work more efficiently by combining real-time, computer-generated environments with production workflows. However, its boundaries and terminology evolve rapidly, making knowledge fragmented across academic papers, conference demos, standards initiatives, vendor documentation, and production case studies.

This project addresses three central research questions:

  • What XR-relevant filmmaking tools and systems have been described across academic and industry sources, and what production stages and use-cases do they support?
  • What attribute dimensions consistently characterize these tools in ways that enable structured comparison?
  • Can an attribute-based similarity metric support navigation of the design space in a scalable way?

Taxonomy & Codebook

Every entry in the database is coded with the following attributes, derived from a systematic analysis of the literature and aligned with established XR and filmmaking taxonomies:

Attribute Type Description
Film Pipeline Stage(s) Multi-label The stage(s) of the film production process: Pre-production, Production, Post-production.
Use-Case(s) Multi-label Specific filmmaking tasks: Previsualization & Storyboarding, Virtual Location Scouting/Set Design, On-Set Visualization (Virtual Production), Immersive Storytelling (XR Content Creation), Remote Collaboration, Training & Rehearsal, Post-production Review.
XR Modality Categorical Type of extended reality technology: AR, VR, or MR.
Platform/Hardware Multi-label Required hardware: VR HMD (Tethered/Standalone), AR/MR Headset, Mobile Device, Projection/Spatial Display, Tracking System, Other Peripherals.
Core Interaction Techniques Multi-label Interaction patterns: Navigation/Travel, Selection & Manipulation, System Control, Annotation/Markup, Collaboration Mechanisms, Perspective/Camera Control.
Input/Output Data Formats Multi-label Data types: 3D Models/Assets, Spatial Scans/Point Clouds, 2D Images/Video, Motion/Tracking Data, Game Engine/Scene Files, Metadata/Script Data.
Target User Role(s) Multi-label Intended users: Director, Cinematographer/DP, Production Designer, VFX Supervisor, Editor, Producer, Actor/Performer, Other Crew.
Evaluation Multi-label Validation methods: User Study, Expert Review, Case Study, Controlled Experiment, Qualitative Analysis, No Formal Evaluation.
Reported Outcomes Multi-label Key results: Efficiency Gains, Improved Decision Quality, Enhanced Collaboration, Higher Usability, Creative Enablement, Realism/Presence, Learning Outcomes.
Open Science Categorical Availability of code/data: No (Closed), Code Available, Data Available, Code & Data Available.
Maturity Ordinal Development stage: Concept → Prototype → Deployed/Pilot → Commercial/Production-Ready.

Similarity Metric

The similarity visualization uses a Gower-style mixed-data distance function that combines Jaccard similarity for multi-label attributes (e.g., pipeline stages, use-cases, modalities) with exact-match similarity for categorical attributes (e.g., maturity, open science) and normalized proximity for numerical attributes (e.g., year). This approach enables automatic similarity computation without requiring expert-created similarity judgments for every pair.

Search Strategy & Methodology

Literature was gathered through a multi-pronged search strategy spanning Google Scholar, ACM Digital Library, IEEE Xplore, arXiv, Springer/Elsevier, SIGGRAPH archives, SMPTE publications, and industry sources (Epic Games, ILM, FXGuide, American Cinematographer, etc.). The search covered 2014–2025, with selective inclusion of earlier foundational works.

The methodology follows a PRISMA-style approach with backward and forward snowballing from a seed list of 46+ key references, iterative query refinement, and deduplication.

Contributing

We welcome contributions from the research community. There are two ways to submit entries or corrections:

  • Email: Submit a new entry via email — include the paper/PDF and details for each codebook attribute listed above.
  • GitHub Issues / Pull Requests: Open an issue or submit a PR adding entries to filmXRVault/data/database.json. Follow the JSON schema above.

All submissions are reviewed for completeness and coding accuracy before inclusion in the database.

Site Architecture

The website is a lightweight static site designed for long-term maintenance:

  • Hosting: GitHub Pages at https://sroberto27.github.io/filmXRVault/
  • Stack: Pure HTML, CSS, and vanilla JavaScript — no build tools or frameworks required
  • Data: Versioned JSON file in the repository (data/database.json)
  • Deployment: Push to main branch → automatic GitHub Pages deployment

File structure:

sroberto27.github.io/
└── filmXRVault/
    ├── index.html              # Main database page
    ├── about.html              # About, codebook, contribution guide
    ├── css/
    │   └── style.css           # All styles
    ├── js/
    │   └── app.js              # Application logic
    ├── data/
    │   └── database.json       # Versioned database
    └── img/                    # (optional) images, thumbnails

Deployment Checklist

  • Clone your existing repository: git clone https://github.com/sroberto27/sroberto27.github.io.git
  • Create a filmXRVault/ folder inside the repository root
  • Copy all project files (index.html, about.html, css/, js/, data/) into filmXRVault/
  • Commit and push to main
  • Visit https://sroberto27.github.io/filmXRVault/ to verify
  • Add DOI via Zenodo integration for data citation (optional)
